Flutter のマテリアル デザインとマテリアル コンポーネントとは何ですか?

マテリアル デザインは、大胆で美しいデジタル製品を構築するためのシステムです。 スタイル、ブランディング、インタラクション、モーションを一貫した一連の原則とコンポーネントの下で統合することにより、製品チームは設計の可能性を最大限に引き出すことができます。

Flutter 用マテリアル コンポーネント (MDC-Flutter) は、デザインとエンジニアリングを、アプリやプラットフォーム全体で一貫したユーザー エクスペリエンスを作成するコンポーネントのライブラリと結び付けます。 マテリアル デザイン システムが進化するにつれて、これらのコンポーネントは更新され、Google のフロントエンド開発基準に準拠して、一貫したピクセル パーフェクトな実装が保証されます。 MDC は、Android、iOS、および Web でも利用できます。

この Codelab では、MDC Flutter のいくつかのコンポーネントを使用してログイン ページを作成します。

TextField ウィジェットを追加する

const SizedBox(height: 120.0) の後に 2 つの新しいテキスト フィールドとスペーサーを追加します。

// TODO: Add TextField widgets (101)
// [Name]
TextField(
  decoration: const InputDecoration(
    filled: true,
    labelText: 'Username',
  ),
),
// spacer
const SizedBox(height: 12.0),
// [Password]
TextField(
  decoration: const InputDecoration(
    filled: true,
    labelText: 'Password',
  ),
  obscureText: true,
),

各テキスト フィールドには、InputDecoration ウィジェットを受け取る decoration: フィールドがあります。

filled: フィールドは、テキスト フィールドの背景が軽く塗りつぶされていることを意味し、ユーザーがテキスト フィールドのタップまたはタッチ ターゲット領域を認識できるようにします。

2 番目のテキスト フィールドの obscureText: true 値は、ユーザーが入力した入力を、パスワードに適した箇条書きに自動的に置き換えます。

TextButton と ElevatedButton の選択

単純に 2 つの ElevatedButton を表示しないのはなぜですか? 各ボタンの種類は、どのアクションが他よりも重要かを示しています。

レイアウトには目立つボタンを 1 つ含める必要があります。 これにより、他のボタンの重要性が低いことが明確になります。 この目立つボタンは、アプリを進めるためにユーザーに最も求めているアクションを表しています。

私たちが彼らに取ってほしくない行動は、ログインをキャンセルすることです。隆起したボタンは、隆起した外観で目を引くため、より重要なアクションに使用する必要があります。 比較すると、その左側にあるプレーン テキスト ボタンはあまり強調されていないように見えます。

マテリアル ガイドラインのボタンの記事で、ボタンの階層について詳しく学んでください。

onPressed: フィールドに空のブロックがあるのはなぜですか?

null を渡した場合、またはフィールドを含めなかった場合 (デフォルトで null になります)、ボタンは無効になります。 タッチに対するフィードバックはなく、有効な動作についてよく理解できませんでした。 空のブロックを使用すると、無効化されなくなります。

このビューを閉じるには、このページ (Flutter がルートを呼び出す) をナビゲーション スタックからポップ (または削除) します。

Navigator は、iOS の UINavigationController と同様にルートのスタックを維持します。 ルートをプッシュすると、スタックの一番上に配置されます。 スタックをポップすると、最後に追加されたルートが削除されます。 アプリの app.dart で、initialRoute: '/login' を呼び出して、ログイン画面をナビゲーターに追加します。

ルートとナビゲーションの詳細については、ナビゲーションの基本をご覧ください。
